# Solution Architect Checklists

## Architecture Quality Checklist

- Clear system boundary.
- Clear module responsibility.
- No duplicated ownership.
- Data flow defined.
- API responsibility defined.
- Error handling defined.
- Security boundary defined.
- Audit logging defined.
- Compliance reviewed.
- Human review points defined.
- MVP scope controlled.

## Healthcare Architecture Checklist

- Patient data protected.
- Clinical AI is decision support only.
- Clinical safety disclaimer included.
- SOAP, diagnosis, and prescription flow traceable.
- Human review for high-risk cases.
- Audit trail for clinical changes.
- Consent and access control considered.

## Insurance Architecture Checklist

- Claim readiness logic traceable.
- Evidence requirement defined.
- Payer rule integration considered.
- Exception handling defined.
- Claim decision is not fully automated.
- Claim reviewer review point defined.
- Audit log for readiness score and rule result.

## AI Architecture Checklist

- AI input/output defined.
- Confidence score handled.
- Low-confidence behavior defined.
- Explainability required.
- Human-in-the-loop defined.
- Prompt/version traceability considered.
- Prohibited AI actions listed.

## Security & Compliance Checklist

- Role-based access considered.
- RLS boundary considered.
- Sensitive data minimized.
- PDPA/OIC impact reviewed.
- Audit log defined.
- Data retention considered.
- Access history considered.
