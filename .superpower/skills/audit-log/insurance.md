# Insurance Audit Rules

## Claim Readiness Score Audit

- Log `CLAIM_READINESS_CALCULATED` whenever a score is calculated or recalculated.
- Capture score, component summary, evidence completeness status, payer rule version references, source module, request ID, and correlation ID.
- Do not represent the score as claim approval or payment guarantee.

## Score Component Audit

- Store structured component summaries such as documentation completeness, ICD consistency, missing evidence count, payer rule warnings, and review status.
- Avoid raw clinical details; reference source documents and evidence records.

## Missing Evidence Audit

- Log `MISSING_EVIDENCE_DETECTED` when required evidence is absent, incomplete, expired, or inconsistent.
- Capture evidence type, source rule, affected claim/visit, severity, and recommended next action.

## Evidence Package Generation Audit

- Log `EVIDENCE_PACKAGE_GENERATED` with package ID, visit ID, included document references, generated by, and generation timestamp.
- Capture warnings for excluded, missing, or stale evidence.

## Evidence Package Export Audit

- Log `EVIDENCE_PACKAGE_EXPORTED` and `PDF_GENERATED` for exports.
- Capture actor, package ID, file/report reference, destination type, export purpose, patient/visit references, and PDPA-sensitive flag.
- Do not log raw exported document contents.

## Payer Rule Evaluation Audit

- Log `INSURANCE_RULE_EVALUATED` with payer rule ID, payer rule version, result, uncertainty, missing evidence references, and correlation ID.
- Log payer rule create/update with reason, before/after fields, actor, and reviewer when applicable.
- Do not invent payer policies or imply coverage when rule evidence is incomplete.

## Claim Override Audit

- Log `CLAIM_READINESS_OVERRIDE` for manual score or status changes.
- Require reviewer identity, role, reason, before value, after value, affected components, and compliance tags.
- Treat unsupported overrides as high risk.

## Coverage Warning Audit

- Log warnings when coverage criteria may be incomplete or inconsistent.
- Use careful wording: warning or decision support only, not final coverage determination.

## Cost Threshold Audit

- Log economic alerts or financial threshold warnings with source rule, threshold reference, and review requirement.
- Avoid exposing unnecessary billing details in broad audit views.

## Reviewer Action Audit

- Log reviewer notes, evidence acceptance/rejection, override decisions, and export actions.
- Preserve final human authority and handoff status.

## Audit Support for Claim Dispute and Payer Review

- Audit trails must connect visit, patient reference, evidence package, claim readiness score, missing evidence, payer rule evaluations, exports, and reviewer actions.
- Events must be filterable by patient, visit, claim package, payer rule, reviewer, date, severity, and category.

## Examples

### 1. Claim Readiness Calculated

- Event category: `CLAIM_READINESS_CALCULATED`
- Actor: `system_claim_readiness`
- Target entity: `visit: vis_demo_2001`
- Required payload: score `82`, component summary, payer rule version references, evidence package status, correlation ID, risk `MEDIUM`, severity `NOTICE`.
- QA note: Verify recalculation creates a new audit event.

### 2. Missing Evidence Detected

- Event category: `MISSING_EVIDENCE_DETECTED`
- Actor: `system_evidence_review`
- Target entity: `evidence_requirement: ev_req_demo_001`
- Required payload: visit ID, evidence type `lab_result_reference`, payer rule reference, missing status, risk `MEDIUM`, severity `WARNING`.
- QA note: Verify event does not store raw clinical documents.

### 3. Evidence Package Exported

- Event category: `EVIDENCE_PACKAGE_EXPORTED`
- Actor: `usr_claim_reviewer_041`
- Target entity: `evidence_package: ep_demo_001`
- Required payload: patient reference, visit ID, report ID, destination type, export reason, `pdpa_sensitive: true`, risk `HIGH`, severity `HIGH`.
- QA note: Verify only authorized roles can export and view export events.

### 4. Manual Claim Override Submitted

- Event category: `CLAIM_READINESS_OVERRIDE`
- Actor: `usr_claim_supervisor_009`
- Target entity: `claim_readiness: cr_demo_001`
- Required payload: before score, after score, changed components, reason, reviewer role, correlation ID, risk `HIGH`, severity `HIGH`.
- QA note: Verify override is rejected if reason is missing.
